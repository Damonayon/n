# SECURITY.md — Политика безопасности проекта

## 🔑 Что считается секретом

| Тип | Где живёт | Как ротировать |
|-----|-----------|----------------|
| `TELEGRAM_BOT_TOKEN` | GitHub Secrets | @BotFather → `/revoke` → новый токен в Secrets |
| `GH_MODELS_TOKEN` | GitHub Secrets | github.com → Settings → PAT → Regenerate |
| `SENTRY_DSN` | GitHub Secrets (опц.) | sentry.io → Project Settings → Client Keys |
| Telegram user_id модератора | GitHub Secrets | не секрет в строгом смысле, но не публикуем |

**Ни один секрет НЕ должен попадать в репозиторий.** Все используется через `os.environ` (в Actions — `${{ secrets.* }}`).

---

## 🚨 Если ты случайно закоммитил секрет

Не паникуй. **Только удаление из репо НЕ помогает** — секрет уже в истории git.

Алгоритм:
1. **Сразу отозвать токен** (на стороне сервиса — Telegram, GitHub, Sentry).
2. Создать новый секрет, обновить GitHub Secrets.
3. (Опционально) удалить секрет из истории через [git filter-repo](https://github.com/newren/git-filter-repo) — но это переписывает историю, force-push обязателен.

## 🛡 Автоматические проверки

Проект использует несколько слоёв защиты:

| Уровень | Инструмент | Когда срабатывает |
|---------|------------|-------------------|
| 1. Поиск утечек | **Gitleaks** ([.github/workflows/security.yml](.github/workflows/security.yml)) | При каждом push/PR и раз в неделю |
| 2. Уязвимости зависимостей | **pip-audit** (там же) | При каждом push/PR |
| 3. Автообновление зависимостей | **Dependabot** ([.github/dependabot.yml](.github/dependabot.yml)) | Каждый понедельник |
| 4. GitHub Secret Scanning | Встроенное GitHub | Постоянно (для публичных репо — бесплатно) |
| 5. Валидация конфига при старте | **pydantic-settings** ([bot/config.py](bot/config.py)) | Каждый запуск |

## 📋 Принципы кода

- **Минимум прав у токенов:** PAT для GH Models даёт только `Models: Read-only`.
- **Изоляция:** все три workflow используют **одни и те же** secrets — нет специальных токенов с расширенными правами.
- **Не логируем секреты:** `bot.logging_setup` НЕ выводит env-переменные в stdout/Sentry.
- **HTTPS only:** все внешние запросы идут через HTTPS (см. `bot/http.py`).
- **Боты — публичные, каналы — публичные/приватные:** доступ к публикации только у владельца бота.
